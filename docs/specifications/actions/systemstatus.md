# System Status Action 规格

状态：已实现
最后代码核对：2026-07-22
action key：`systemstatus`
UUID：`com.ulanzi.ulanzistudio.lexutility.systemstatus`

## 1. 功能定位

System Status 在一个普通方键上持续显示本机系统指标。设置中可从 CPU 占用、RAM 占用、GPU 占用、CPU 温度、上传速度、下载速度六项中选择 1～3 项；显示顺序与设置槽位一致。

本 action 采集并展示当前值，同时为每种指标保留最近 24 个有效采样点，用于键面微型趋势图；不执行压力测试，也不把硬件信息发送到外部服务。

## 2. 数据源与平台边界

- 通用采样层使用 Node.js 进程内的跨平台系统信息接口，覆盖 CPU、RAM 与网络接口累计流量。
- Windows 的 GPU 占用与 CPU 温度优先读取本机 LibreHardwareMonitor HTTP API（默认 `http://127.0.0.1:8085/data.json`）；未运行时回退跨平台采样层。
- macOS 的 GPU 占用与 CPU 温度使用跨平台采样层可获得的系统探针；系统或权限不提供读数时显示 `N/A`，不得伪造 0。
- 网络速度由连续两次累计字节数的差值计算，首次采样显示等待态；聚合所有有效物理接口，排除 internal/loopback。
- 所有探测仅访问本机；LibreHardwareMonitor URL 只接受 loopback HTTP 地址。

## 3. 设置契约

| 字段 | 默认值 | 归一化 | 含义 |
| --- | --- | --- | --- |
| `metric1` | `cpu` | 六种指标之一 | 第一行 |
| `metric2` | `ram` | 六种指标之一、`none` 且不得重复 | 可选第二行 |
| `metric3` | `download` | 六种指标之一、`none` 或不得重复 | 可选第三行 |
| `pollSec` | `2` | 1..30 秒 | 采样间隔 |
| `lhmUrl` | `http://127.0.0.1:8085/data.json` | loopback HTTP URL | Windows 高级传感器来源 |
| `theme` | `signal` | 公共主题 | 通用配色 |
| `frameSize` | `optimal` | `optimal` / `max` | 安全框尺寸 |
| `showFrame` | `true` | 字符串布尔 | 是否显示公共边框 |

Inspector 以三个有序下拉槽位表达“最多 3 项”；第二、第三项可选“无”，第二项关闭时第三项随之关闭。归一化会去重、前置排列并保证至少一项有效指标。

## 4. 生命周期

| 钩子 | 行为 |
| --- | --- |
| `createState` | 创建当前实例的采样状态与网络基线，并从 `action-state.json` 安全水合最近 24 条历史 |
| `onReady` | 立即采样并登记实例定时器 |
| `onRun` | 取消待执行的轮询计时并手动立即采样；采样期间点亮系统标志，完成后恢复并重新安排轮询 |
| `onSettingsChanged` | 采样间隔或数据源变化时重置网络基线并重新调度 |
| `onDispose` | 同步补刷尚未落盘的历史；不发起新探测，定时器由框架统一回收 |
| `render` | 依据归一化 settings + 当前 state 纯函数生成 SVG data URL |

异步采样使用每实例的 in-flight 标记防止重叠；定时器只通过 `setInstanceTimeout` 登记。实例销毁后采样结果不得回写或重绘。手动刷新若与自动采样重叠，不丢弃按键请求，而是在当前采样后紧接着排队执行一次手动采样；重复请求最多合并为一个待执行刷新。每种指标只记录有限数值且单独截断到最近 24 条；不可用读数不写成 0。历史属于运行态，使用框架 `readPersistedState` / `writePersistedState` 与设置分离；累计 24 个采样批次才写盘一次，实例释放时补刷，禁止逐采样写盘。历史缺失、版本不匹配或内容损坏时安全退化为空历史。

## 5. 键面显示

- 全部内容经公共 `renderThemeBackdrop`、`frameFor`、`frameContent` 输出，复用 `THEMES`。
- 顶部左上角显示系统标志，右侧显示 `SYSTEM` 标题：macOS 使用苹果矢量标志，Windows 使用视窗矢量标志；不显示 `LIVE`、`SCAN`、`WAIT`、`STALE` 等采样状态胶囊。
- 手动按键刷新期间，系统标志由主题 `accent` 提亮为主文字色，并显示低透明度圆形光晕；反馈至少保持 300ms，且必须等采样完成后才恢复原色。自动轮询不显示该反馈。
- 主区显示 1～3 行指标卡；CPU、RAM、GPU、温度、上传、下载均使用约 20～22px 的独立线性矢量图标，不再显示 `CPU`、`RAM`、`TEMP`、`NET` 等文字标签。每行同时保留主数值和单位，字号随行数调整但主数值不小于 24px。
- 原有单值背景进度条统一替换为最近 24 条趋势图：CPU、RAM、GPU、CPU 温度默认使用柱状图，上传与下载默认使用折线图；图表置于指标卡背景层，使用当前语义色的中低透明度版本，不遮挡图标、数值与单位。
- 百分比柱状图固定按 0～100 缩放，温度柱状图以 110°C 为正常上界并允许更高实值扩展；网络折线图按各自最近 24 条记录的峰值动态缩放。历史不足 24 条时从右侧逐步填充。
- 温度与网络速度使用同一行高和对齐体系。上传/下载图标以独立矢量上下箭头区分方向；`KB/s`、`MB/s` 等单位固定使用 15px、主题 `accent` 色，在不挤占主数值的前提下保持清晰区分。
- 正常强调色取主题 `accent`；不可用显示 `N/A` 与 `muted`；探测失败保留上次有效值，不在头部额外显示运行状态。

## 6. 验证要求

- 六种指标枚举、1～3 项去重与归一化有单元测试。
- CPU/RAM/网络差分及 LibreHardwareMonitor 树解析使用固定夹具测试，不依赖测试机实时数值。
- 1/2/3 行渲染均输出合法 SVG，不含 `undefined` / `NaN`，且主数值字号满足下限。
- 历史追加、24 条截断、安全水合与序列化有单元测试；柱状图/折线图默认映射及 24 点 SVG 结构有渲染测试。
- macOS / Windows 左上角标志及采样状态文字移除有双平台结构测试；六类指标的矢量图标和趋势图加深后的透明度有结构测试。
- 手动刷新测试覆盖立即高亮、采样完成后恢复、轮询重新调度，以及与自动采样重叠时排队补刷。
- manifest、action 注册、Inspector、静态图标与本规格保持一致。
- 完整 `npm test` 通过；因新增 action identity 和主进程模块，桌面验收使用 `restart`。

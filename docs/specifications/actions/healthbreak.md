## 1. 功能定位

Health Break 面向长期使用电脑的程序员、主播、影视编辑等人群，以短而频繁、无需器械、容易完成的微活动打断久坐与持续屏幕暴露。一个物理键位是一套独立健康方案；同一个 action 可以拖入多个键位，每个实例独立配置动作组、周期、每日目标、有效时段、声音和统计。

本 action 只提供一般健康习惯提醒，不诊断疾病、不验证动作质量，也不替代医疗建议。统计中的“完成”只表示用户完整跟完一次引导。

四层实现目标：

- manifest：`manifest.json` 的 `Health Break`。
- 业务：`plugin/actions/healthbreak.js`。
- Inspector：`property-inspector/healthbreak.html`、`healthbreak.js`。
- 静态图标：`assets/icons/actionHealthbreak.svg`。

## 2. 动作库与组合

首版内置六个动作组：

| key | 名称 | 默认内容 | 声音/视觉语义 |
| --- | --- | --- | --- |
| `eyes` | 护眼 | 远眺、主动眨眼 | 远眺扩散图、眨眼节拍 |
| `neck` | 颈肩 | 下巴微收、缓慢左右转头、肩胛后收 | 方向提示与阶段切换音 |
| `hands` | 手腕手指 | 握拳张开、温和腕部伸展、放松 | 张合/方向提示 |
| `stand` | 起身活动 | 起身、提踵、原地走动 | 站立/步进节拍 |
| `breathe` | 呼吸复位 | 放松肩膀、缓慢吸气与呼气、坐姿复位 | 吸/呼阶段音 |
| `pelvic` | 盆底肌 | 短收缩与充分放松交替 | 收缩/放松阶段音 |

- 每个实例必须选择 1..4 个动作组，默认 `eyes,neck`。
- 每轮按用户在 Inspector 中确定的顺序完整执行全部已选组。
- 首版只允许组合内置动作，不开放任意动作文本、上传声音或逐动作时长编辑。
- 盆底肌默认不选；首次选用须确认“盆底肌并非越紧越好，疼痛、排尿排便困难、术后或孕产期问题应先咨询专业人士，练习必须包含充分放松”。
- 动作均以 20 秒到约 1 分钟的原地微活动为主，无器械、不追求出汗。疼痛、眩晕或麻木时应立即停止。

## 3. 设置契约

| 字段 | 默认值 | 合法范围/选项 | 含义 |
| --- | --- | --- | --- |
| `groups` | `eyes,neck` | 六个 group key 中去重后的 1..4 项 | 动作组及执行顺序 |
| `intervalMin` | `45` | 5..240 分钟 | 有效用电脑时间周期 |
| `dailyGoal` | `6` | 1..12 | 每个健康日的目标完成次数 |
| `activeStart` | `09:00` | `HH:MM` | 有效时段起点 |
| `activeEnd` | `18:00` | `HH:MM` | 有效时段终点；小于起点表示跨午夜 |
| `activeDays` | `0,1,2,3,4,5,6` | 0..6 去重列表 | 以有效时段开始日计算的星期 |
| `repeatReminderMin` | `5` | 0..30 分钟 | 到期后的增强提醒间隔；0 关闭周期增强 |
| `soundEnabled` | `true` | 字符串布尔值 | 到期、阶段与完成提示音总开关 |
| `theme` | `mint` | 公共主题 key | 外观 |
| `frameSize` | `optimal` | `optimal` / `max` | 安全显示范围 |
| `showFrame` | `true` | 字符串布尔值 | 是否绘制公共边框 |

Inspector 提供动作组选择与排序、周期、目标、时段、星期、提醒与声音设置、主题、安全框、保存和恢复默认。动作详情与安全说明放在 Inspector；键面只承担最简引导、当前提示与计时/次数。

## 4. 健康日、有效时段与活动时间

- 普通时段（如 09:00–18:00）按自然日运行。
- 跨午夜时段（如 20:00–03:00）整体归属开始日期，午夜不重置目标。
- 时段开始时创建新的健康日记录并从完整周期开始；上一时段欠下的提醒不带入。
- 时段外、系统睡眠、锁屏或长期无用户输入时暂停周期累计；恢复后继续剩余周期，不立即补发过期提醒。
- 睡眠通过 tick 间隔跳变识别；锁屏/长期无人使用通过平台活动探测尽力识别。平台探测失败时降级为普通有效时段计时，不阻断 action。
- 已经开始的练习可跨越有效时段结束并计入刚结束的健康日；尚未处理或静默排队的提醒在时段结束时自动作废，不计跳过。

## 5. 状态机与按键语义

主要状态：

```text
waiting -> due -> running -> waiting/done
             \-> waiting              长按跳过
due/queued -> queued                   其他实例占用提醒或练习资源
running <-> paused                     短按
running/paused -> waiting/done         完成
running/paused -> waiting              长按取消
done -> running -> done                主动加练
```

- `waiting` / 时段外 / `done` 短按：主动开始一轮完整练习。
- `due` 短按：开始本轮；长按：跳过此次提醒，跳过不计达标并从完整周期重新计时。
- `running` 短按：暂停；`paused` 短按：继续。
- `running` / `paused` 长按：取消本轮，不计达标并从完整周期重新计时。
- 非到期、非练习状态的长按不执行 Health Break 业务。
- 尚未到期的主动练习完成后计入目标并重置周期；达标后的主动练习记为加练，不恢复自动提醒。
- 最后一个动作结束后自动计为完成，不需要二次确认。

## 6. 跨实例协调与提醒

- Health Break 全部实例同一时间只允许一套练习运行，避免计时和声音叠加。
- 多个实例到期时，最早到期者进入 `due` 并闪烁，其余为静默 `queued`。
- 当前练习完成、跳过或取消后保留 2 分钟提醒缓冲，再激活最早到期实例。
- 当前练习中短按另一实例时，目标实例进入手动优先队列；当前练习结束后自动开始，不打断当前动作。
- 到期前 2 分钟使用明显呼吸闪烁，之后低频闪烁；每逢 `repeatReminderMin` 周期的短窗口恢复明显闪烁。持续到开始、跳过、时段结束或实例被移除。
- 到期只播放一次短提示音；练习开始、动作组/阶段切换和完成按动作语义播放短提示。暂停立即停止后续节拍，恢复后从当前阶段重新对齐。

## 7. 计时与运行态持久化

- 周期与练习阶段都以“剩余有效活动毫秒数”为事实源，只在有效时段且用户活动时扣减。
- 实例 tick 使用基座 `setInstanceTimeout`，禁止裸 `setTimeout` / `setInterval`。
- 运行态写入 `data/action-state.json`，结构版本为 `v: 1`，至少保存：
  - 当前健康日 key、周期剩余量、状态与到期时间；
  - 当前动作组/阶段索引及剩余量；
  - 当日完成、加练、跳过、取消；
  - 最近 30 个健康日的聚合记录。
- 运行态在状态转换、健康日切换、设置重算和 dispose 时同步落盘；长期等待时可按分钟级检查点批量落盘，不逐秒写磁盘。
- 重启恢复时，进行中的练习恢复为暂停，防止无人确认时自动继续；损坏、版本不符或非法字段降级为空。

## 8. 键面显示

- `waiting`：显示当前实例全部 1..4 个动作组图标、距下次提醒的紧凑倒计时、今日 `完成/目标`。
- 时段外：保持动作组图标，倒计时位置显示 `OFF`。
- `due`：显示首个动作组与 `开始`，按提醒强度呼吸闪烁。
- `queued`：显示动作组与 `等待`，不闪、不响。
- `running`：显示当前动作的极简方向/节奏动画、简短提示和剩余秒数或次数。
- `paused`：冻结当前引导并叠加暂停标记。
- `done`：显示完成勾与简洁庆祝状态；主动加练完成后短暂显示 `加练 +1` 再返回 done。

所有颜色来自公共 theme token，内容经 `frameContent` 放入安全框，不新增私有主题或绕过公共画布契约。

## 9. 统计口径

- 完整跟完一轮：`completed +1`；达到 `dailyGoal` 后停止当天自动提醒。
- 达标后的主动练习：`bonus +1`。
- 到期长按：`skipped +1`；练习中长按：`cancelled +1`。
- 暂停后继续完成仍正常计入；未完整完成不计达标。
- 按钮只显示今日进度；Inspector 展示最近 30 天完成、加练、跳过、取消、是否达标与连续达标天数。
- 数据仅保存在本地，不记录每次操作的精确时间线，不上传。

## 10. 平台与降级

- macOS 与 Windows 为正式支持目标；Linux 保持核心计时、统计和动画可用。
- macOS 提示音使用系统播放器，Windows 使用 PowerShell beep，Linux/其他平台回退终端 bell或静音。
- 活动探测、声音或平台命令失败只关闭对应增益能力，不得让 action 进入 ERR。
